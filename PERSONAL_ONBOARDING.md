**System Builder · Fiduciary Architect · Bare-Metal Native**

## 1. Core Computing Lineage (Pre-Cloud, Pre-GUI)

My technical root system did not start with tutorials, platform hand-holding, or abstraction.  
In elementary school I assembled, powered, and operationalized legacy IBM/ACC-era hardware:  
- No instructions  
- No adult guidance  
- No abstraction layers  
- Direct power-on → functional system

This bred a physical ↔ logical intuition:
- I understand systems by interacting, not by reading about them.
- I expect determinism and clear cause/effect.
- I distrust “magic layers” unless I can deconstruct them.

### This is why:
- CLI feels native.
- Filesystems are tangible.
- Permissions, ports, tokens, and keys are levers, not mere concepts.

---

## 2. Mental Model: Systems, Not Tools

I think in terms of infrastructure and state—not apps or products.
- Cloudflare: network edge
- Terraform: state enforcement, not “magic apply”
- GitHub: versioned memory, not “code host”
- SSH: operational presence

> I separate:  
> Identity vs credentials  
> Tokens vs authority  
> State vs configuration  
> Public surface vs private control plane

This is not recent learning, but innate muscle memory revived from early computing.
